## Gaps (Stream 2 specific)

- **Databricks** (named explicitly, "extensive experience... using Databricks") — **no evidence
  anywhere in the wiki.** This is the single biggest gap and borderline deal-breaker if interpreted
  literally as hands-on Databricks delivery. Severity: high but survivable — the JD is a *project
  manager* role (not a data engineer), and the surrounding asks (governance, integration, ingestion,
  reporting, cross-agency data sharing) are all delivery-management competencies Alan has repeatedly
  demonstrated on other platforms. Bridge: acknowledge directly, pivot on the "technical domain gap +
  delivery-focus pivot" device already used for Optimizely/Enterprise Historian/SCADA gaps — the
  platform is new, the job of running governance/schedule/risk/vendor coordination around a data
  platform delivery is not.
- **Cross-agency data sharing / external government agencies** — no direct evidence. Bridge: Xero's
  cross-domain integration coordination (dozens of teams, federated API surface) and Suncorp's
  cross-brand/cross-vendor coordination are the closest available analogues, but this is a genuine new
  context (government-to-government, not enterprise-internal).
- **Named data/security compliance frameworks specific to government (IRAP, information security
  classification, etc.)** — not evidenced. Suncorp's "data governance & compliance" work is
  APRA/financial-services-flavoured, not government information-security-flavoured. Acknowledge as
  adjacent, don't overclaim.
